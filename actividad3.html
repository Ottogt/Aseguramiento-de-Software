PASO 3: Crear Rama DEV a partir de la Rama Principal

Se crea una nueva rama llamada "DEV" a partir de la rama principal (main). Esta rama contendrá los cambios adicionales del número de carnet.
Comandos Ejecutados:
$ git branch DEV
$ git branch -a
Salida del Git:
DEV * main remotes/origin/HEAD -> origin/main remotes/origin/main
✓ Resultado: Rama DEV creada exitosamente. Se puede ver que existen dos ramas locales: main y DEV

PASO 4: Agregar Número de Carnet en Rama DEV

Se cambia a la rama DEV y se agrega el número de carnet (08001620) al archivo config.txt.
Comandos Ejecutados:
$ git checkout DEV
$ echo "08001620" >> config.txt
$ git add config.txt
$ git commit -m "Paso 4: Agregar número de carnet en rama DEV"
$ cat config.txt

PASO 5: Pull Request para Unificar Ramas

Se hace push de la rama DEV al repositorio remoto y se crea un Pull Request para unificar los cambios con la rama principal.
Comandos Ejecutados:
$ git push -u origin DEV
$ git log --all --oneline --graph
Salida del Git:

✓ Resultado: Rama DEV publicada en el repositorio remoto. Pull Request disponible en: Crear Pull Request
